# Check-list Code Review (минимальный)

## Общие принципы
- [ ] Объём PR разумный; цель понятна (одна тема на PR)
- [ ] Изменения покрыты тестами (позитив/негатив)
- [ ] Обработка ошибок и граничных случаев
- [ ] Именование и структура кода понятны
- [ ] Док/комментарии релевантны и точны

## Безопасность
- [ ] Нет хардкода секретов/паролей
- [ ] Валидация входных данных
- [ ] Обработка исключений без утечки информации
- [ ] Использование безопасных практик (например, для работы с БД)

## Производительность
- [ ] Нет очевидных узких мест
- [ ] Разумное использование ресурсов
- [ ] Отсутствие лишних циклов/запросов

## Тестирование
- [ ] Покрытие основных сценариев
- [ ] Тесты на граничные случаи
- [ ] Тесты на обработку ошибок
- [ ] Интеграционные тесты (если применимо)

## Документация
- [ ] README обновлен (если нужно)
- [ ] Комментарии в коде понятны
- [ ] API документация актуальна

## Рекомендации ревьюеру
- Комментируйте **по делу** и по строкам/диффу, избегайте «LGTM» без аргументов
- Используйте *Request changes* для блокирующих замечаний
- Старайтесь оставлять **конкретные** предложения исправлений
- Проверяйте, что CI проходит после изменений
