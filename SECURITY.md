# Security Policy

## Сообщение об уязвимостях

Если вы обнаружили уязвимость в этом проекте, пожалуйста, сообщите об этом:

- **Контакт**: mixail.rychagov@student.hse.ru
- **Срок ответа**: 48 часов
- **Канал**: приватное сообщение или email

## Политика безопасности

- **НЕ** храните ключи/секреты в репозитории
- **НЕ** коммитьте файлы с паролями, API ключами или токенами
- Используйте синтетические данные для тестирования (без ПДн/платежей)
- Все ошибки возвращаются в едином JSON формате (см. README)
- При работе с внешними API используйте переменные окружения для конфиденциальных данных

## Обработка инцидентов

1. Немедленно удалите секреты из кода/истории
2. Инвалидируйте скомпрометированные ключи в провайдере
3. Сообщите преподавателю о произошедшем
4. Обновите документацию при необходимости
